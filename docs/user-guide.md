# 小红书卡片生成器使用指南

## 这个工具是做什么的

小红书卡片生成器可以把 Markdown 文章转换成适合小红书发布的竖版图片卡片。

你可以用它处理：

- 技术文章
- 学习笔记
- 博客草稿
- Typora 写好的 Markdown
- 普通 `.md` / `.markdown` 文件

工具会自动完成：

```text
读取 Markdown
-> 拆成多张卡片
-> 套用主题样式
-> 预览效果
-> 导出 PNG 图片
```

## 使用前准备

你需要安装 Chrome、Edge 或 Chromium 中的任意一个。

原因是工具导出 PNG 时，会在本地调用浏览器截图。所有内容都在你的电脑本地处理，不会上传到服务器。

## 下载后如何使用

### macOS

1. 解压 `XHS Cardgen-mac.zip`
2. 双击 `XHS Cardgen.app`
3. 浏览器会自动打开工具页面
4. 选择 Markdown 文件
5. 点击“预览”
6. 确认效果后点击“导出 PNG”

### Windows

1. 解压 `XHS Cardgen-win.zip`
2. 双击 `start.bat`
3. 浏览器会自动打开工具页面
4. 选择 Markdown 文件
5. 点击“预览”
6. 确认效果后点击“导出 PNG”

## 页面上的设置是什么意思

### Markdown 文件

选择你的 `.md` 或 `.markdown` 文件。

也可以直接把文章内容粘贴到 Markdown 内容框里。

### 标题

可选。

如果不填，工具会自动读取文章里的一级标题。

### 底部署名

显示在每张卡片底部的小字。

适合填写：

- 专栏名
- 账号名
- 文章主题

### 主题皮肤

当前内置多种主题：

- 暖色技术风
- 清爽白底
- 深色代码风
- 薄荷杂志风
- 技术黑金
- 极简纸感
- 杂志标题风
- 日系手账
- 产品发布会风
- 学术笔记风

可以先切换主题，再点击“预览”查看效果。生成预览后继续切换主题，也会直接用当前卡片内容刷新预览，不会重新解析 Markdown。

### 每页字数

控制每张卡片的大致文字密度。

建议：

- 内容太挤：调小
- 卡片太多、空白太多：调大

### 每页块数

控制每张卡片最多放多少个内容块。

内容块包括：

- 段落
- 列表
- 代码块
- 表格
- 引用
- 图片

建议：

- 页面溢出：调小
- 空白太多：调大

### 输出文件夹路径

可选。

不填时，工具会默认保存到桌面上的 `xhs-cardgen-exports` 文件夹。

### Chrome 或 Edge 路径

通常不用填。

工具会自动检测常见安装位置。如果导出失败并提示找不到 Chrome/Edge，可以手动填写浏览器路径。

## 推荐使用流程

1. 选择 Markdown 文件。
2. 选择主题。
3. 点击“预览”。
4. 在“卡片内容编辑”里微调标题、导语、正文或页脚。
5. 根据需要删除、复制、上移或下移卡片。
6. 如果卡片太多，调大“每页字数”或“每页块数”，也可以直接删除不需要的卡片。
7. 如果内容太挤，调小“每页字数”或“每页块数”。
8. 点击“导出 PNG”。
9. 打开输出文件夹，把图片上传到小红书。

## 如何手动优化生成内容

预览生成后，左侧会出现“卡片内容编辑”区域。

你可以：

- 选择某一张卡片
- 修改顶部标签、标题、导语、正文块和页脚
- 删除不需要的卡片
- 复制一张卡片继续改
- 上移或下移卡片顺序
- 点击“应用修改”刷新预览

手动修改的是生成后的 `slides.json`，不会改写你的原始 Markdown 文件。

如果你已经做过手动修改，再点击“预览”重新从 Markdown 生成，工具会提示这些修改将被覆盖。

### 不同内容块如何编辑

- 段落、引用、代码：直接在文本框里编辑。
- 无序列表、有序列表、标签、流程：每行代表一项。
- 表格：第一版使用 JSON 编辑，适合高级用户微调。
- 图片：第一版只支持查看和修改图片路径、说明文字，不做图片替换。

## 小红书图片数量限制

小红书单篇图文最多发布 18 张图片。

如果工具提示超过 18 张，你可以：

- 调大每页字数
- 调大每页块数
- 把文章拆成上下篇

## 如果提示溢出怎么办

导出时，工具会检测哪些卡片可能有内容溢出。

如果看到类似提示：

```text
第 3、7 张可能溢出
```

建议你：

1. 降低“每页字数”
2. 降低“每页块数”
3. 重新点击“预览”
4. 再次导出 PNG

当前版本不会自动改写文章，也不会自动删除内容。所有分页都由规则控制，你可以通过参数调整结果。

## 是否会调用 AI 大模型

不会。

当前版本不会调用 LLM，也不会把你的文章上传到任何在线模型。它只在本地做：

```text
Markdown 解析
规则分页
HTML/CSS 渲染
本地浏览器截图
```

## 输出文件里有什么

导出后，输出目录通常包含：

```text
slides.json
preview.html
cards/
  card-01.png
  card-02.png
  ...
```

其中：

- `cards/` 是可以直接上传小红书的图片。
- `preview.html` 是可在浏览器打开的预览页面。
- `slides.json` 是中间结构文件，方便开发者或高级用户继续处理。

## 常见问题

### 为什么要安装 Chrome 或 Edge

因为卡片是用 HTML/CSS 排版的，导出 PNG 时需要浏览器渲染并截图。

### 为什么预览很快，导出稍慢

预览只生成 HTML。

导出需要启动本地 Chrome/Edge，把每张卡片截图成 PNG，所以会慢一些。

### 为什么有些代码块比较长

工具会尽量保留完整代码块，避免把一段代码拆成很多碎片。太长的代码仍可能被拆页。

### 是否支持 Typora

支持。

Typora 保存的 Markdown 本质上也是 Markdown。普通 `.md` 和 Typora `.markdown` 都可以使用。

如果文章里有本地图片，后续版本会继续增强图片路径处理。
